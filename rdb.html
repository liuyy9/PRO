一、产品综述

RDB（Report Data Bridge，报表数据桥）是一款企业级 ETL 数据集成工具，可将业务系统中分布的、异构数据源中的数据（如关系型数据、数据文件等）抽取到临时中间层后进行清洗、转换与集成，最终加载到数据仓库或数据集市中，为联机分析处理、数据挖掘及辅助决策提供统一、可靠的数据基础。同时引入 AI 智能能力，大幅提升 ETL 配置与运维效率。

RDB 通过可视化流程编排、全面的元数据管理、数据治理能力、智能运维监控与 AI 智能辅助，为企业提供从数据接入到数据治理的全链路解决方案。

二、核心能力

01可视化流程编排

支持拖拽式图形化流程配置与展示，内置 47 类 ETL 核心组件，覆盖数据导入、导出、转换、传输、质量校验、实时消息对接等全类型操作，分钟级完成复杂数据处理流程的设计与发布。

02全面的元数据管理

提供完善的 RDB 元数据配置体系，支持 14 类对象元数据类型管理，涵盖数据库对象、表对象、SQL 脚本、存储过程、数据文件、Hadoop 相关对象、RestAPI 对象等。

03数据治理能力

内置数据血缘分析与数据质量管理双引擎，支持 SQL 血缘关系自动解析与可视化展示，以及数据一致性、波动性、有效性、完整性四维数据质量校验。

04智能运维监控

提供全面的流程运行状态监控，支持历史任意时间范围内的流程实例补跑、无延时流程调度，以及组件报错自动重处理机制，显著降低运维干预成本。

05AI 智能辅助

内置专属 RDB 知识库，提供交互式问答、流程配置辅助、复杂流程解读及 ETL 故障根因分析四大 AI 能力，助力配置与运维人员大幅提升工作效能。

三、功能特性

3.1 WorkFlow Manager — 可视化流程管理

提供图形化的 ETL 流程设计与管理能力，支持拖拽式配置、流程测试与发布、批量激活等功能。

  • 拖拽式图形化流程配置与展示
  • 流程的测试与发布，确保上线前充分验证
  • 流程的批量激活，提升批量运维效率

3.2 WorkFlow Monitor — 流程运行监控

提供全面的 ETL 流程运行状态监控与管理能力，支持历史补跑与无延时调度。

  • 展示流程实例统计信息与实时统计
  • 支持任意历史时间范围的流程实例补跑
  • 支持无延时流程调度，保障实时性要求

3.3 Repository Management — 元数据配置管理

支持 RDB 流程运行所需元数据对象的统一配置与管理，目前支持 14 类元数据对象类型。

  • DB 对象、Table 对象、SQL 对象
  • Hadoop 相关对象、RestAPI 对象
  • 全面代表 RDB 的能力范围

3.4 RDB 核心组件库

平台目前支持 47 类 ETL 核心组件，覆盖数据导入导出、文件操作、质量校验等全场景。

  • 不同类型数据源的数据导入与导出
  • 数据质量校验与实时消息对接
  • 流程控制与机器学习组件

3.5 Data Governance — 数据治理

包含数据血缘管理（DATA FamilyTree）与数据质量管理（DQ Management）双引擎。

  • 自动解析 SQL 生成血缘关系图谱
  • 数据一致性、波动性、有效性、完整性校验
  • 血缘关系高亮链路可视化显示

3.6 Agent of Workflow — 流程权限管控

实现不同用户对同一流程的增删改查权限的精细化控制，满足多人协作开发场景。

  • 精细化权限控制
  • 多人协作开发支持
  • 流程资产安全保障

3.7 Exception Alarm Configuration — 异常自动重处理

实现组件报错自动重处理功能，减少运维人员对常规故障的人工介入。

  • 组件报错自动重处理
  • 减少常规故障人工介入
  • 提升系统稳定性与运维效率

3.8 Message Monitor — 实时消息监控

提供 MQ 实时消息监控及流水查看功能，适用于实时数据同步场景。

  • MQ 实时消息监控
  • 消息流水查看
  • 实时数据同步场景支持

3.9 AI 能力

平台内置专属 RDB AI 智能体，提供四大 AI 辅助能力，助力提升工作效能。

  • 交互式知识问答
  • 流程配置辅助与复杂流程解读
  • ETL 故障根因分析

四、适用场景

4.1 多源异构数据集成

适用于将 Oracle、MySQL、HIVE、ClickHouse 等多种异构数据源统一集成至数据仓库的场景。

4.2 数据仓库定时批量加载

适用于企业数据仓库的定时批量 ETL 任务场景，确保数据按时准确入仓。

4.3 实时数据同步与消息处理

适用于对接 MQ 消息队列实现实时数据同步的场景，保障消息数据的可靠处理。

4.4 数据质量监控与治理

适用于对报表数据质量有严格要求的场景，通过多维度校验保障数据可信度。

4.5 数据血缘追溯与合规审计

适用于需要追溯数据来源与加工链路的数据治理场景，满足合规与审计要求。

4.6 多团队协作 ETL 开发

适用于多开发人员协作维护 ETL 流程的场景，通过权限管控保障流程资产安全。

4.7 ETL 故障快速响应与修复

适用于对数据处理稳定性要求较高的生产环境，显著缩短故障平均修复时间。
